# Tiger run — 2026-06-20 (baseline)

First hunt. Inventoried every LLM call site into the vault, scored each on code-quality + business-value, and benchmarked two quality-sensitive tools across model tiers. This is the baseline future scans diff against.

## Headline
The **text plumbing is gold-standard** (one chokepoint with retry/fallback/self-repair/cost-stamp/telemetry — [[llm-wrapper]]), and the analytics tools are **richly grounded** in real numbers. The value is leaking at the **edges**: (1) no response **caching** on `/api/ai` (every identical call re-pays), (2) the three **non-text** calls (image/vision/embedding) are **completely un-instrumented and uncached** — the priciest call in the app (Leonardo) is invisible, (3) two **dropped/optional grounding** paths quietly starve good prompts, and (4) **model tier matters**: the copy/reply tools need ≥ Sonnet; the constrained tools are the untested downgrade prize.

## Impact-ranked backlog (frequency × reach × value)

### P0 — highest leverage
- **V1 · [[lp-variant-ideas]] dropped-grounding bug** (value+code). The UI computes and POSTs `controlCvr` + `losers`, but `validateLpVariantIdeasRequest` (`validation.ts`) never copies them onto `value`, so the prompt's two strongest signals never arrive (grounding **2/5**). The registry probe also omits them, so it's invisible to CI. **Fix:** thread both fields through the validator → 2/5 → 4/5. One-line-ish fix, unblocks [[david-cro-experiments]]'s entire JTBD.
- **C1 · No input-hash cache on `/api/ai`** (code/efficiency). All ~12 text tools recompute identical inputs every call; [[campaign-eval]] already proves the pattern (`hashEvalInputs` + `findCachedReport` + quota-only-on-real-call). **Fix:** port it to `/api/ai`. Cleanest wins first: [[analysis]], [[keyword-clusters]], [[cohort-diagnosis]], [[lead-source-diagnosis]] (deterministic/stable inputs).
- **C2 · Non-text calls are un-instrumented** (code/logging). [[creative-image-gen]] (Leonardo — most expensive call), [[creative-vision-score]] (N Gemini vision calls/gen), [[patterns-embed]] bypass the wrapper and record **zero** cost/latency/tokens. **Fix:** wire a `recordLlmCall`-equivalent so one telemetry dashboard covers every modality.

### P1 — major
- **C3 · [[patterns-embed]] re-embeds the whole corpus every search** (`M×(N+1)` calls). Precompute + persist vectors on pattern-save; embed only the query. O(N)→O(1)/search.
- **C4 · [[lead-reply]] & [[local-review-reply]] have no `validate`** — a blank model reply is silently floored to canned text with no telemetry flag (failure reads as success). Add a `validate` (non-empty + structure) so the wrapper self-repairs and the miss is observable.
- **V2 · [[social]] via `/api/ai` omits grounding + brand** (only `/api/social/draft` passes them) → degrades ~4/5 → ~2/5. Pass grounding+brand on the `/api/ai` path or deprecate it.
- **M1 · Model floor for copy/reply tools** — benchmark says [[lead-reply]] + [[ads]] **need ≥ Sonnet** (Haiku drops specificity / violates char limits). Since prod runs Gemini Flash, **quality-check Flash on these two; consider Gemini Pro** for them. See [[benchmark-2026-06-20]].

### P2 — polish / next-pass
- **V3 · brand voice into [[lead-reply]]/[[local-review-reply]]** so "on-brand" is grounded (pass business name+voice) — branded, not generic (ties to UAT's Marta finding).
- **V4 · always pass `peers` to [[lead-source-diagnosis]]** (4/5→5/5 grounded peer comparison).
- **C5 · cap [[repurpose]]'s article-body input** (only tool with an unbounded prompt).
- **C6 · golden snapshots** for the 5 tools that have only a live probe (cohort-diagnosis, lead-source-diagnosis, lead-reply, local-review-reply, keyword-clusters/etc.).
- **M2 · benchmark the CONSTRAINED tools on Haiku** ([[keyword-clusters]], [[analysis]], [[cohort-diagnosis]], [[lead-source-diagnosis]]) — hypothesised safe downgrade = real cost win. Add the thinking-level axis.
- **C7 · lower temperature** on strictly-grounded reads ([[analysis]] 0.7 → ~0.4).
